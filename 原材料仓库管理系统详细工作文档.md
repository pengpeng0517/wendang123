## 一、项目概述

### 一.一 项目背景

本课题源于鞋类制造业原材料仓库管理现状的深入调研。鞋类原材料涵盖皮革、布料、胶料、鞋带等多种类型，这些物料存在种类繁多、规格复杂、数据记录不准确、信息传递滞后、追溯困难等诸多问题。针对这些痛点，迫切需要设计开发一套高效、精准的仓库管理系统。传统仓库普遍采用固定规则分配库位，即按照物料品类划分存储区域。这种静态分配方式存在明显弊端：某些区域易出现爆仓而其他区域长期闲置，导致库位利用率低下；同时拣货路径冗长、效率低下。因此，本课题在设计基础仓库管理功能的同时，引入基于强化学习的动态库位分配模块，实现库位的自适应优化，显著提升仓库运营效率。

鞋类原材料仓库管理效率的高低直接影响企业的生产成本和交货周期。传统人工数据录入方式不仅容易出错，而且难以实现全程追溯；同时缺乏智能预警机制，库位分配策略僵化不变。随着鞋企规模持续扩大、原材料SKU数量急剧增加，传统管理方式已经难以满足现代化仓库管理的需求。SpringBoot框架凭借其快速开发、微服务支持、自动配置等显著特性，为构建轻量级仓库管理系统提供了坚实的技术基础。此外，强化学习在组合优化领域展现出强大的应用潜力，能够自主学习最优决策策略。因此，开发融合SpringBoot与强化学习技术的鞋类原材料仓库管理系统具有重要的现实意义和应用价值。

### 一.二 国内外研究现状

#### 一.二.一 国外研究现状

国外仓库管理系统研究起步较早，自二十世纪八十年代开始发展，经历了从简单库存记录到智能化管理系统的演变。现代仓库管理系统已高度集成射频识别技术、物联网、大数据分析等先进技术。美国Dematic公司开发的智能仓储系统实现了全自动化物料搬运和库存管理，被Nike、Adidas等国际知名鞋企广泛应用；德国Schenker物流公司推出的仓库管理系统支持多仓库协同管理，能够处理复杂的鞋类原材料供应链网络。日本在精益生产理念指导下，开发了适合鞋类制造业的即时制原材料管理系统，实现了原材料的精准配送。欧洲鞋类制造商普遍采用SAP扩展仓储管理模块，实现与企业资源计划系统的深度集成，支持原材料批次追踪、效期管理和质量检验。当前国外仓库管理系统发展趋势是更加智能化、集成化和云化，通过人工智能算法优化库存布局，预测原材料需求，提高仓库空间利用率和作业效率。

#### 一.二.二 国内研究现状

我国仓库管理系统的研究始于二十世纪九十年代，早期主要依赖进口系统或简单自研工具。二零零零年后，随着国内制造业快速发展，国产仓库管理系统逐渐兴起，但多集中于通用仓储场景，针对鞋类原材料特殊性的研究相对较少。国内研究主要集中在以下几个方面：一是基于云平台的轻量级仓库管理系统开发，降低中小企业使用门槛；二是结合射频识别、条码技术实现自动化数据采集；三是利用大数据分析优化库存管理策略；四是针对特定行业如鞋服行业的专业仓库管理系统研发。近年来，京东物流、菜鸟网络等企业推出的智能仓储解决方案，为行业提供了新的技术参考。在鞋类行业，安踏、李宁等国内领先企业已开始引入智能化仓库管理系统，国产自主可控的系统仍有较大发展空间。基于SpringBoot的轻量级仓库管理系统因其开发周期短、部署灵活、维护成本低等特点，逐渐成为中小企业实施数字化转型的优选方案。

#### 一.二.三 现有系统存在的问题

当前仓库管理系统在鞋类原材料管理领域存在以下主要问题：

行业针对性不足，现有系统多为通用型设计，未能充分考虑鞋类原材料的特殊属性，如材质差异、季节性需求波动、存储条件要求等，导致系统功能与实际业务需求脱节。系统集成度低，多数系统与企业资源计划系统、制造执行系统等对接不完善，形成信息孤岛，数据无法有效流通和共享，严重影响整体运营效率。智能化程度有限，库存预警、需求预测等功能较为简单，缺乏基于历史数据和市场趋势的智能分析能力，无法为企业决策提供有效支持。用户体验欠佳，尤其针对仓库现场操作人员，界面设计不够友好，操作流程繁琐，增加了操作人员的学习成本和工作负担。实施成本偏高，大型仓库管理系统价格昂贵，部署周期长，维护成本高，不适合中小型鞋企的实际需求和预算约束。库位分配策略僵化，现有系统多采用固定规则分配库位，无法有效适应鞋类原材料季节性剧烈波动，导致库位利用率低或频繁出现爆仓情况，严重影响仓库运营效率。

### 一.三 项目目标

本课题的主要目的是设计并实现基于SpringBoot框架的鞋类原材料仓库管理系统，实现入库管理、出库管理、库存管理、预警管理等功能，同时引入强化学习动态库位分配模块，实现库位的智能优化。具体目标包括以下几个方面：

第一，建立原材料唯一标识体系实现全程追踪。通过设计科学的编码规则和分类体系，为每种原材料建立唯一的标识编码，支持全程追溯和精准查询，确保物料流转的每一个环节都有据可查。这不仅有助于提升库存数据的准确性，更能为质量追溯提供可靠的数据支撑，当出现质量问题时能够快速定位问题批次，最大限度降低损失。

第二，实现库存实时监控与管理。构建实时库存管理系统，动态更新库存数据，支持多维度库存查询和分析功能，为管理决策提供及时准确的数据支持。实时监控能够确保管理人员随时掌握库存动态，及时发现异常情况并采取相应措施，避免因库存不足导致的生产中断或库存积压造成的资金占用。

第三，设计智能预警机制。建立基于生产计划的安全库存模型，当库存低于安全库存或超过最大库存时自动触发预警通知，有效避免原材料短缺影响正常生产或库存积压占用仓储空间。预警机制应支持多级预警设置，针对不同重要程度的物料设置不同的预警阈值，实现差异化管理。

第四，构建完善的权限管理体系。基于角色访问控制模型设计用户权限体系，实现细粒度的访问控制，确保数据安全和操作合规。不同角色的用户只能访问其职责范围内的功能和数据，所有敏感操作都需要经过授权验证，防止未授权访问和数据泄露风险。

第五，提供数据报表分析功能。生成各类统计报表，支持库存盘点、入出库分析、库存周转等业务分析功能，为企业管理层提供科学的数据支撑和决策依据。报表应支持多种图表展示方式，直观呈现数据趋势和异常情况，帮助管理层快速把握业务全貌。

第六，设计并实现基于强化学习的动态库位分配模块。通过深度分析库存状态和物料属性特征，强化学习算法能够自主学习最优库位分配策略，动态调整存储位置，有效降低爆仓风险，显著提升拣货效率，优化仓库空间利用率。这一创新模块是本课题的核心技术亮点，体现了人工智能技术在传统仓储管理领域的创新应用。

## 二、需求分析

### 二.一 用户角色定义

本系统用户角色基于实际业务需求设计，采用基于角色访问控制的权限管理模型，确保不同角色的用户只能访问其职责范围内的功能和数据，实现精细化的权限管理。用户角色定义如下表所示：

| 角色名称 | 角色描述 | 主要职责 |
|----------|----------|----------|
| 系统管理员 | 负责系统配置、用户管理和权限分配 | 系统参数配置、用户账号管理、角色权限分配、数据备份恢复 |
| 仓库管理员 | 负责仓库日常运营和库存管理 | 入库操作、出库操作、库存盘点、库位管理、报表统计 |

#### 二.一.一 角色权限矩阵

本系统功能模块的权限分配如下表所示：

| 功能模块 | 系统管理员 | 仓库管理员 |
|----------|------------|------------|
| 用户管理 | 全部权限 | 无访问权限 |
| 角色管理 | 全部权限 | 无访问权限 |
| 原材料管理 | 查看、新增、修改、删除 | 查看、新增、修改 |
| 入库管理 | 全部权限 | 全部权限 |
| 出库管理 | 全部权限 | 全部权限 |
| 库存管理 | 全部权限 | 全部权限 |
| 库位管理 | 全部权限 | 全部权限 |
| 报表统计 | 全部权限 | 全部权限 |
| 系统配置 | 全部权限 | 无访问权限 |

### 二.二 功能需求规格

#### 二.二.一 用户管理模块

用户管理模块是系统安全运行的基础保障，提供完整的用户认证、授权和权限管理功能。该模块采用JWT令牌实现无状态认证，基于角色访问控制模型实现细粒度的权限控制，确保系统资源和数据的安全。模块功能需求如下表所示：

| 功能名称 | 功能描述 | 优先级 | 输入 | 输出 |
|----------|----------|--------|------|------|
| 用户登录 | 提供用户身份认证，支持账号密码登录方式，验证用户凭证并生成JWT令牌 | 高 | 用户名、密码 | JWT令牌、用户信息 |
| 用户信息管理 | 支持用户的增删改查操作，维护用户基本信息和状态 | 高 | 用户信息表单 | 操作结果、用户列表 |
| 角色权限管理 | 支持角色的创建、编辑、删除和权限分配，实现细粒度访问控制 | 高 | 角色信息、权限列表 | 角色列表、操作结果 |
| 密码管理 | 支持密码修改和密码重置功能，保障用户账户安全 | 中 | 旧密码、新密码 | 操作结果 |

### 四.一 用户管理模块设计

#### 四.一.一 模块概述

用户管理模块是系统安全运行的基础保障，提供完整的用户认证、授权和权限管理功能。该模块采用JWT令牌实现无状态认证，基于角色访问控制模型实现细粒度的权限控制，确保系统资源和数据的安全。模块设计遵循高内聚低耦合的原则，各功能组件职责清晰，便于维护和扩展。

#### 四.一.二 功能流程设计

用户登录流程如下：用户输入用户名和密码，系统验证用户凭证，验证通过后生成JWT令牌并返回给客户端，客户端在后续请求中携带该令牌进行身份认证。用户注销流程：客户端请求注销接口，系统使令牌失效并清除服务端会话状态。权限验证流程：系统根据用户角色和资源权限配置，判断用户是否有权访问指定资源，无权访问时返回禁止访问错误。

#### 四.二.五 物料编码生成算法

物料编码采用"MAT"前缀加三位数字的格式（如MAT001、MAT002），生成算法如下：

1. 查询数据库中物料编码的最大值
2. 解析最大编码中的数字部分
3. 新编码数字部分为最大值加1
4. 格式化为"MAT%03d"格式

**代码实现：**
```java
private String generateMaterialCode() {
    LambdaQueryWrapper<Material> wrapper = new LambdaQueryWrapper<>();
    wrapper.orderByDesc(Material::getCode);
    wrapper.last("LIMIT 1");
    Material maxMaterial = materialMapper.selectOne(wrapper);
    
    int nextNumber = 1;
    if (maxMaterial != null && maxMaterial.getCode() != null) {
        String code = maxMaterial.getCode();
        if (code.startsWith("MAT")) {
            try {
                int currentNumber = Integer.parseInt(code.substring(3));
                nextNumber = currentNumber + 1;
            } catch (NumberFormatException e) {
                nextNumber = 1;
            }
        }
    }
    return String.format("MAT%03d", nextNumber);
}
```

**算法特点：**
- 基于最大编码值生成，确保编码连续
- 删除物料后，新物料编码自动填补空缺
- 支持最多999个物料（MAT001-MAT999）
- 编码格式统一，便于识别和排序

**后端权限控制**：
- 在`OutboundOrderController`中实现`isSystemAdmin()`方法判断当前用户是否为系统管理员
- 从Spring Security的`SecurityContextHolder`中获取当前认证信息
- 解析JWT令牌获取用户名，通过`UserService`查询用户详细信息

3. 验证出库单是否存在且状态为"待审批"
4. 获取出库单的明细数据，遍历明细减少对应物料的库存数量
5. 如果用户是系统管理员，更新出库单状态为"已审批"并返回成功信息
6. 如果用户不是系统管理员，返回权限不足错误信息
7. 前端根据返回结果显示相应的提示信息

**权限验证流程（删除操作）**：
1. 用户点击删除按钮，前端发送DELETE请求到`/api/outbound/order/delete/{id}`
2. 后端控制器调用`isSystemAdmin()`方法验证用户权限
3. 验证出库单是否存在且状态为"待审批"（已审批的出库单不允许删除）
4. 如果用户是系统管理员，删除出库单及其明细记录并返回成功信息
5. 如果用户不是系统管理员，返回"只有系统管理员可以删除出库单"错误信息
6. 前端根据返回结果显示相应的提示信息

**双重权限保障**：
- 前端控制：通过Vue.js条件渲染控制按钮显示，非管理员用户看不到删除按钮
- 后端控制：在控制器中严格验证用户权限，防止通过直接调用API绕过前端限制
- 这种双重保障机制确保只有系统管理员才能执行敏感操作（删除、批准出库单）

### 四.五 库存管理模块设计

#### 四.五.一 模块概述

库存管理模块是系统的核心业务模块，负责物料的入库、出库、盘点、预警等操作，实现库存的全生命周期管理。该模块确保库存数据的准确性和实时性，为企业运营提供可靠的库存数据支持。模块支持多维度的库存查询和统计分析，满足不同角色的数据需求。

#### 四.五.二 功能流程设计

**库存查询流程**：用户进入库存管理页面，系统展示所有物料的库存列表，包括物料编码、物料名称、规格、单位、库存数量、单价、金额等信息。支持按物料编码、物料名称等关键词搜索，支持查看库存详情和库存流水记录。

**库存更新流程**：
- **入库更新**：当入库记录被创建时，系统自动检查库存表中是否存在该物料的库存记录。如果存在，则增加库存数量；如果不存在，则创建新的库存记录，从物料表获取物料信息（编码、名称、规格、单位、单价等），设置初始库存数量为入库数量。
- **出库更新**：当出库单被批准时，系统自动减少对应物料的库存数量。如果库存不足，则返回错误信息。出库成功后生成库存流水记录。

**库存流水记录流程**：每次库存变动（入库或出库）时，系统自动生成库存流水记录，记录变动类型、数量、操作人、操作时间等信息，支持库存变动的追溯和审计。

**库存预警流程**：系统实时监控库存变动，当库存低于安全库存或超过最大库存时自动触发预警通知，推送给相关责任人处理。

#### 四.五.三 类设计

库存管理模块核心类设计如下表所示：

| 类名 | 类型 | 描述 |
|------|------|------|

#### 四.五.五 库存同步机制设计

**入库同步机制**：
- 当创建入库记录时，StorageServiceImpl调用InventoryService的addStock方法
- addStock方法首先检查库存表中是否存在该物料的库存记录
- 如果存在，则更新库存数量（原数量 + 入库数量）
- 如果不存在，则从物料表获取物料信息，创建新的库存记录，设置初始数量为入库数量
- 同时生成库存流水记录，记录入库操作

**出库同步机制**：
- 当出库单被批准时，OutboundOrderServiceImpl调用InventoryService的reduceStock方法
- reduceStock方法检查库存是否充足
- 如果库存充足，则减少库存数量（原数量 - 出库数量）
- 同时生成库存流水记录，记录出库操作
- 如果库存不足，则返回错误信息，出库审批失败

**数据一致性保障**：
- 入库和出库操作使用Spring事务管理，确保数据一致性
- 库存更新和流水记录生成在同一个事务中完成
- 如果任一操作失败，整个事务回滚，保证数据完整性

### 四.七 智能库位分配模块设计

#### 四.七.一 模块概述

智能库位分配模块是本系统的核心技术创新点，基于强化学习算法实现库位的动态优化分配。该模块通过分析库存状态和物料属性，自主学习最优库位分配策略，显著提升库位利用率和拣货效率。模块采用深度Q网络或近端策略优化等强化学习算法，通过与仓库环境的交互学习最优库位分配决策。

#### 四.七.二 强化学习模型设计

状态空间设计：状态空间包括当前库存状态、各库位占用情况、入库物料属性、历史出入库数据等特征。系统将库存状态编码为高维向量，库位信息包括位置、容量、当前存放物料类型等信息，物料属性包括尺寸、重量、周转率、出入库频率等特征。

动作空间设计：动作空间为所有可用库位的选择，通过候选库位筛选机制缩小动作空间，提高学习效率。对于每种入库物料，系统从符合存储条件的库位中筛选出候选库位集合，智能体从候选库位中选择最优库位。

奖励函数设计：奖励函数综合考虑库位利用率、拣货距离、物料相关性、库位均衡等因素。系统定义多目标奖励函数，将多个优化目标加权组合为单一奖励值，指导智能体学习最优策略。奖励函数设计需要平衡多个目标，避免单一目标过度优化导致其他目标恶化。

#### 四.七.三 类设计

智能库位分配模块核心类设计如下表所示：

| 类名 | 类型 | 描述 |
|------|------|------|
| RLEnvironment | 类 | 强化学习环境，模拟仓库环境 |
| RLAgent | 类 | 强化学习智能体，实现学习算法 |
| LocationRecommender | 类 | 库位推荐服务，封装推荐逻辑 |
| RLConfig | 配置类 | 强化学习算法配置参数 |

```java
// 基于最大编码值生成新编码
private String generateMaterialCode() {
    LambdaQueryWrapper<Material> wrapper = new LambdaQueryWrapper<>();
    wrapper.orderByDesc(Material::getCode);
    wrapper.last("LIMIT 1");
    Material maxMaterial = materialMapper.selectOne(wrapper);
    
    int nextNumber = 1;
    if (maxMaterial != null && maxMaterial.getCode() != null) {
        String code = maxMaterial.getCode();
        if (code.startsWith("MAT")) {
            try {
                int currentNumber = Integer.parseInt(code.substring(3));
                nextNumber = currentNumber + 1;
            } catch (NumberFormatException e) {
                nextNumber = 1;
            }
        }
    }
    return String.format("MAT%03d", nextNumber);
}
```

## 六、系统实现细节

### 六.一 后端开发规范

#### 六.一.一 项目初始化

使用Spring Initializer创建SpringBoot项目，添加Web、MySQL、Redis、Security等必要依赖。配置Maven镜像加速依赖下载，设置项目编码为UTF-8，配置Lombok简化代码编写。创建主启动类和配置文件，完成项目基础框架搭建。

#### 六.一.二 分层架构实现

Controller层负责接收和响应HTTP请求，进行参数校验和结果封装，不包含业务逻辑。Service层负责业务逻辑的实现和事务管理，调用Repository层进行数据访问。Repository层负责数据访问操作，使用MyBatis Plus提供的通用Mapper减少样板代码。Entity层负责数据实体的定义，与数据库表对应。

#### 六.一.三 统一响应封装

定义统一的响应结果类Result，包含状态码、消息、数据等字段。定义响应状态枚举ResultCode，包含成功、业务错误、系统错误等状态码。封装成功和失败的响应方法，便于控制器层快速返回响应结果。

### 六.二 安全认证实现

#### 六.二.1 JWT认证流程

用户登录成功后，服务端生成JWT令牌，令牌包含用户ID、角色、过期时间等信息。令牌采用HS256算法签名，服务端保存密钥用于验证令牌有效性。客户端在后续请求中通过Authorization请求头携带令牌，服务端过滤器拦截请求，解析并验证令牌，验证通过后设置安全上下文。

#### 六.二.2 权限控制实现

使用Spring Security框架实现方法级别的权限控制。通过@PreAuthorize注解在方法上配置权限表达式，只有拥有指定权限的用户才能调用该方法。定义权限枚举类Permission，包含各功能模块的权限常量。实现权限校验服务，根据用户角色和资源权限配置进行校验。

### 六.三 业务功能实现

#### 六.三.一 原材料管理实现

原材料编码采用分类前缀加流水号的方式自动生成，确保编码的唯一性。分类管理支持无限级分类，通过递归算法构建分类树。供应商管理支持供应商信息的新增、编辑、查询、禁用等操作。

#### 六.三.二 入库管理实现

入库单支持采购入库、退货入库等多种类型，通过类型字段区分。入库流程包括创建、提交、审核、确认等状态流转，使用状态机模式管理状态变化。入库确认时自动更新库存数量和库位状态，生成库存流水记录。

#### 六.三.三 出库管理实现

出库单支持生产领料、销售出库等多种类型。出库流程包括创建、审批、确认等状态流转。库存扣减采用乐观锁机制，防止并发操作导致的数据不一致。出库确认时更新库存数量和库位状态，释放被锁定的库存。

#### 六.三.四 库存管理实现

库存查询支持多条件组合查询和分页，使用Redis缓存热点数据提高查询性能。库存预警采用定时任务扫描库存表，发现异常库存后发送预警通知。库存盘点支持全盘和抽盘两种方式，盘点差异经过审批后调整库存。

#### 六.三.五 报表统计实现

报表数据通过SQL聚合查询或内存计算生成，支持日、周、月、年等时间维度的统计。图表展示采用ECharts组件，支持折线图、柱状图、饼图等多种图表类型。数据导出采用Apache POI库生成Excel文件。

### 六.四 智能库位分配实现

#### 六.四.一 算法模型训练

使用PyTorch框架搭建深度Q网络或近端策略优化模型，定义神经网络结构。在仿真环境中训练智能体，通过大量模拟入库场景学习最优策略。保存训练好的模型参数，用于在线推理推荐。

#### 六.四.2 推荐服务实现

库位推荐服务接收入库请求和物料信息，构建状态向量输入模型。模型输出各候选库位的评分，选择评分最高的库位作为推荐结果。推荐服务封装为RESTful接口，供入库模块调用。

## 九、性能优化

### 九.1 后端优化

#### 九.一.一 数据库优化

合理设计索引，对高频查询字段创建索引。优化SQL语句，避免全表扫描和低效连接查询。使用读写分离分担数据库压力。定期分析执行计划，优化慢查询。

#### 九.一.二 缓存优化

使用Redis缓存热点数据，减少数据库访问压力。合理设置缓存过期时间，保证数据一致性。使用分布式锁防止缓存击穿和缓存雪崩。监控缓存命中率，优化缓存策略。

#### 九.一.三 接口优化

使用分页查询减少单次返回数据量。实现接口级别的超时控制和熔断降级。使用异步处理提高接口响应速度。对耗时操作进行优化，如批量处理、延迟处理。

### 九.二 前端优化

#### 九.二.一 构建优化

使用Vite的Tree Shaking消除无用代码。配置代码分割，按路由懒加载组件。使用Gzip压缩静态资源。优化图片资源，使用WebP格式和懒加载。

#### 九.二.二 运行优化

合理使用Vue computed和watch，避免不必要的计算。使用v-show代替v-if减少组件销毁重建。使用虚拟列表优化长列表渲染。减少重排重绘，优化DOM操作。

**库存同步实现：**
```java
@Override
@Transactional
public boolean addStock(Long materialId, Integer quantity, String operator) {
    Inventory inventory = getInventoryByMaterialId(materialId);
    if (inventory == null) {
        // 从物料表获取物料信息，创建新的库存记录
        Material material = materialService.getById(materialId);
        if (material != null) {
            inventory = new Inventory();
            inventory.setMaterialId(materialId);
            inventory.setMaterialCode(material.getCode());
            inventory.setMaterialName(material.getName());
            inventory.setSpec(material.getSpec());
            inventory.setUnit(material.getUnit());
            inventory.setQuantity(quantity);
            inventory.setPrice(material.getPrice() != null ? material.getPrice() : BigDecimal.ZERO);
            inventory.setAmount(inventory.getPrice().multiply(BigDecimal.valueOf(quantity)));
            inventory.setLocation("");
            inventory.setMinStock(material.getMinStock() != null ? material.getMinStock() : 0);
            inventory.setMaxStock(material.getMaxStock() != null ? material.getMaxStock() : 0);
            inventory.setStatus(1);
            inventory.setCreateTime(LocalDateTime.now());
            inventory.setUpdateTime(LocalDateTime.now());
            save(inventory);
        } else {
            return false;
        }
    } else {
        inventory.setQuantity(inventory.getQuantity() + quantity);
        inventory.setUpdateTime(LocalDateTime.now());
        updateById(inventory);
    }
    
    // 记录库存流水
    InventoryRecord record = new InventoryRecord();
    record.setInventoryId(inventory.getId());
    record.setMaterialId(materialId);
    record.setMaterialCode(inventory.getMaterialCode());
    record.setMaterialName(inventory.getMaterialName());
    record.setType("入库");
    record.setQuantity(quantity);
    record.setPrice(inventory.getPrice());
    record.setAmount(inventory.getPrice().multiply(BigDecimal.valueOf(quantity)));
    record.setOperator(operator);
    record.setCreateTime(LocalDateTime.now());
    inventoryRecordService.addRecord(record);
    
    return true;
}
```

**出库审批实现：**
```java
@Override
@Transactional
public OutboundOrder approveOrder(Long id) {
    OutboundOrder order = getOrderById(id);
    if (order == null || order.getStatus() != 0) {
        return null;
    }
    
    // 获取出库明细，减少库存
    if (order.getDetails() != null && !order.getDetails().isEmpty()) {
        for (OutboundDetail detail : order.getDetails()) {
            if (detail.getMaterialId() != null && detail.getQuantity() != null) {
                inventoryService.reduceStock(
                    detail.getMaterialId(), 
                    detail.getQuantity(), 
                    order.getRecipientName()
                );
            }
        }
    }
    
    order.setStatus(1);
    order.setUpdateTime(LocalDateTime.now());
    updateById(order);
    
    return order;
}
```
