# 基于强化学习的动态库位分配系统

## 1. 系统目标

### 1.1 动态优化
- 根据仓库实时状态和物料属性，为每个入库物料自动推荐最优存储库位
- 支持在库位的动态调整（如移库优化）

### 1.2 多目标平衡
- **降低爆仓风险**：避免局部区域过度拥挤
- **提升拣货效率**：缩短拣货路径，提高作业效率
- **优化空间利用率**：提高仓库存储密度，减少空间浪费

### 1.3 自适应能力
- 能够应对旺季业务量变化
- 自动调整分配策略以适应不同业务场景

## 2. 整体架构

```
┌─────────────────┐     ┌─────────────────────────────────────┐
│    WMS 核心     │     │          RL 库位推荐模块             │
│  - 入库请求     │────▶│  - 状态构建器                        │
│  - 库位状态     │     │  - 候选库位筛选器                    │
│  - 物料信息     │     │  - RL 策略网络                       │
│  - 订单数据     │     │  - 奖励计算器（训练时）              │
└─────────────────┘     └─────────────────────────────────────┘
        ▲                            │
        │                            ▼
        └──────────────────── 推荐库位 / 移库指令
```

### 2.1 模块说明

| 模块 | 功能描述 |
|------|----------|
| WMS 核心 | 处理入库请求，管理库位状态，维护物料信息和订单数据 |
| 状态构建器 | 将WMS数据转换为RL模型可理解的状态向量 |
| 候选库位筛选器 | 从所有空闲库位中筛选出符合条件的候选集 |
| RL 策略网络 | 根据状态选择最优库位 |
| 奖励计算器 | 在训练阶段计算奖励值，评估决策质量 |

## 3. RL 模型设计

### 3.1 状态空间

状态应全面反映仓库当前情况，设计为包含以下分量的向量：

| 维度 | 描述 | 表示方式 |
|------|------|----------|
| **全局仓库状态** | 整体库存量、空闲库位总数、当前爆仓风险指数（如占用率>阈值的巷道数） | 归一化数值 |
| **区域热度** | 各区域（如巷道、货架）的订单需求量（过去N小时出库次数） | 向量，长度=区域数 |
| **物料属性** | 待分配物料的体积、重量、品类、周转率（历史出库频率） | 归一化向量 |
| **候选库位特征** | 每个候选库位的剩余容量、所在区域热度、与热门区域的接近度、高度/承重限制等 | 矩阵 (候选数 × 特征数) |
| **时间特征** | 当前时刻（小时、星期）、是否旺季标志 | one-hot 或归一化 |

### 3.2 动作空间

原始动作空间为所有空闲库位（可能数万个），直接学习困难。因此采用候选库位筛选后，将动作空间缩小到 **K** 个候选库位（如 K=100）。

**动作定义**：从 K 个候选库位中选择一个最优库位。

### 3.3 奖励函数

奖励函数需权衡多个目标，设计为加权和：

```
R = w₁R_efficiency + w₂R_space + w₃R_risk + w₄R_stability
```

#### 3.3.1 各项奖励说明

| 奖励项 | 符号 | 说明 |
|--------|------|------|
| **拣货效率奖励** | R_efficiency | 推荐库位与当前热门区域的接近程度（负的行走距离）。可计算该库位到未来N小时可能出库的热点区域的距离，或直接使用库位所在区域的热度。 |
| **空间利用率奖励** | R_space | 鼓励紧凑存储，例如剩余空间碎片化程度低，或选择能最大化填充率的库位。 |
| **爆仓风险惩罚** | R_risk | 如果选择某个库位会导致某个区域占用率超过阈值，给予负奖励。也可计算全局占用率均衡度。 |
| **稳定性奖励** | R_stability | 鼓励尽量少移动已有库存（如果涉及移库决策），惩罚频繁变动。 |

#### 3.3.2 权重调整

权重 wᵢ 可根据业务重点调整：
- **平时模式**：均衡各项目标
- **旺季模式**：增加 R_risk 的权重，优先防止爆仓
- **促销模式**：增加 R_efficiency 的权重，优先拣货效率

## 4. 候选库位筛选：缩小动作空间

### 4.1 基于规则的初步过滤

#### 4.1.1 物料-库位匹配
- 根据物料体积筛选合适大小的库位
- 根据物料重量筛选承重合格的库位
- 根据存储类型（如危险品、冷藏）筛选符合要求的库位

#### 4.1.2 区域约束
- 某些品类必须存放在指定区域
- 特殊物料需要隔离存储

### 4.2 基于学习的候选生成

#### 方法1：启发式打分
- 使用简单的启发式（如库位热度、剩余容量）对所有可行库位快速打分
- 取 Top-K 作为候选集

#### 方法2：图神经网络（GNN）编码
- 将仓库库位视为节点，构建邻接图
- 使用GNN对所有库位生成嵌入
- 通过评分网络（如MLP）给出每个库位的"潜在价值"
- 取 Top-K 作为候选
- 该评分网络可与RL策略网络联合训练

#### 方法3：注意力机制
- 将全局状态作为Query
- 所有库位特征作为Key
- 计算注意力分数
- 取分数最高的K个库位

**筛选出的候选库位集合构成动作空间，RL模型只需从中选择最优的一个。**

## 5. 训练方法

### 5.1 离线训练（基于历史数据或仿真）

#### 5.1.1 构建仿真环境
- 使用历史入库、出库数据搭建仓库模拟器
- 能够重置状态并执行分配动作
- 模拟后续出库并计算奖励

#### 5.1.2 数据驱动训练
- 若无法仿真，可使用历史分配记录进行离线强化学习（如CQL、BCQ）
- 需要奖励标签

#### 5.1.3 算法选择

| 算法 | 适用场景 | 特点 |
|------|----------|------|
| **DQN** | 离散动作空间 | 基础算法，易于实现 |
| **DDQN** | 离散动作空间 | 解决Q值过估计问题 |
| **Dueling DQN** | 离散动作空间 | 分离状态价值和动作优势 |
| **PPO** | 连续动作空间 | 若动作空间连续化（先选区域再选具体库位） |
| **DRQN** | 变长候选集 | 处理动态变化的候选库位集合 |
| **Transformer** | 变长候选集 | 处理变长候选集，注意力机制 |

### 5.2 在线微调

- 将离线训练好的模型部署到WMS
- 记录每次决策后的真实结果（如后续拣货耗时、爆仓事件）
- 定期收集数据用于在线学习或定期重训

## 7. 旺季策略调整

### 7.1 权重自适应

在旺季，动态增大奖励函数中 R_risk 的权重，使模型更倾向于选择能平衡区域占用的库位，防止局部爆仓。

| 模式 | R_efficiency | R_space | R_risk | R_stability |
|------|-------------|---------|--------|-------------|
| 平时 | 0.3 | 0.3 | 0.2 | 0.2 |
| 旺季 | 0.2 | 0.2 | 0.5 | 0.1 |
| 促销 | 0.5 | 0.2 | 0.2 | 0.1 |

### 7.2 候选筛选调整

- 旺季可扩大候选库位数量 K
- 调整启发式规则，如优先考虑高层货架以释放底层空间

### 7.3 多模型切换

预训练多个针对不同流量模式的模型：
- 平时模型
- 旺季模型
- 大促模型

根据预测流量自动切换。

### 7.4 元学习

训练一个元模型，能够根据当前流量特征快速调整策略（如通过上下文学习）。
